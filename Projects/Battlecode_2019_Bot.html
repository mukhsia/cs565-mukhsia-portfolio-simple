Battlecode 2019 Bot

Description:

Developed a javascript Battlecode 2019 bot with an aggression-focused strategy in a development team of size three. Used an Agile-like development process with 2-week sprints for 4 sprints total, playing against a reference bot at the end of each sprint.

Responsibilities:
  • Setup and manage Slack and Trello board. Integrate Github for Slack, Trello for Slack, Travis CI, and Coveralls
  • Setup Travis CI and Coveralls for automated build and code coverage report generation
  • Wrote batch script for building and testing on Windows environment
  • Implemented Bot Combat helper functions (filter visible units on distance, team, (un)attackable range, etc.)
  • Implemented Bot communication helper function (Report enemy Castle destruction & Communicate next target location)
  • Implemented Bot movement helper functions (Calculate enemy castle location, determine direction, etc.)
  • Initial naive farthest valid move movement implementation
  • Crusader and Prophet bot design
  • Defender, Attacker, and Destroyer sub-role design and implementations for Crusader and Prophet bots
  • Implemented Attacker sub-role behaviors towards attacking enemy castles and units en-route
  • Implemented Defender sub-role behavior of guarding the castle against enemy bots
  • Added role switching from defender to attacker when the number of nearby allied Crusader/ Prophet bots exceeds a certain threshold
  • Implemented Destroyer sub-role behavior to move and secure neutral resources, eliminating and preventing additional enemy resource gathering attempts
  • Designed and implemented Crusader-vs-Prophet, Crusader-vs-Crusader and Prophet-vs-Crusader micro strategies based on visible enemies, engagement distance, and movement speed
  • Write unit tests using mocha, chai, and a battlecode 2019 game mock framework developed by another team member
